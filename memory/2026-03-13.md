# 2026-03-13 工作日志

## Bug 修复：简历下载优化（第二轮修正）

### 问题
用户请求特定候选人简历时（如"田丽琴简历发我"），系统会下载 20 封邮件再过滤，导致性能慢。

### 根因
- `find_unread_candidate_by_name()` 默认 `limit=20`
- 先下载 20 封未读邮件 → 再按姓名过滤

### 第一轮修复（有问题）
- 改为 limit=5
- **问题**：如果候选人不前 5 封未读邮件里，就找不到了

### 最终修复
1. **core/imap_client.py** - `search_header_items_by_name()`:
   - 先用 IMAP 搜索所有符合条件的邮件 UID（不限数量）
   - 按 UID 逐个获取 Header
   - limit 只影响返回结果数量，不影响搜索范围

2. **core/pipeline_ops.py**:
   - `limit` 改为 `None`（返回所有匹配）

3. **core/query_ops.py**:
   - 移除 `limit=5` 参数，搜索所有未读邮件

### 效果
- 搜索所有未读邮件，按姓名精确匹配
- 找到匹配后返回（可能多封，但只会处理第一封）
- 不再盲目下载 20 封只取 1 封

## 候选人简历查询流程（7步优先级）

当用户说"把 xxx 简历发我"时的完整查询顺序：

1. **查 processed 结果**（本地已处理过的候选人库）
   ```python
   candidates = find_candidates_by_name(records, candidate_name, jd_title)
   ```

2. **查 recent_uid 缓存**（最近一次操作的候选人）
   ```python
   exact_recent_local = find_local_download_by_uid(recent_uid)
   ```

3. **查 header index 未读**（本地索引的未读邮件）
   ```python
   index_unread = find_in_header_index(header_index, candidate_name, seen=False)
   ```

4. **查 header index 已读**（本地索引的已读邮件）
   ```python
   index_seen = find_in_header_index(header_index, candidate_name, seen=True)
   ```

5. **查未读邮件**（IMAP 实时搜索）
   ```python
   unread_matches = find_unread_candidate_by_name(candidate_name)
   ```

6. **查已读邮件**（IMAP 实时搜索）
   ```python
   seen_matches = find_seen_candidate_by_name(candidate_name)
   ```

7. **查本地下载目录**（兜底）
   ```python
   local_download = find_local_download_by_name(candidate_name)
   ```

### Header Index 说明
- 索引文件位置：`runtime_dir/state/mail-header-index.json`
- 构建方式：一次性读取所有 UNSEEN 和 SEEN 邮件的 Header（UID、发送者、主题、日期）
- 每个条目带有 `seen` 字段：`False`=未读，`True`=已读
- 搜索索引是内存操作，无需联网，比 IMAP 快很多
